# 출석부 화면 스펙 (AttendancePage)

## 파일 위치
`src/apps/pages/AttendancePage.tsx`

## 화면 목적
집계/통계 없이 **순수하게 주차별 출석을 입력하고 저장하는 화면**.
집계 데이터는 별도 대시보드로 분리.

---

## 화면 구조

```
┌─────────────────────────────────────────────────────┐
│  교구: [1교구 ▼]   팀: [1팀 ▼]   그룹: [전체 ▼]      │
├─────────────────────────────────────────────────────┤
│        [ < ]  2026년 3월 15일 (토)  [ > ]            │
├─────────────────────────────────────────────────────┤
│  이름   기수   출석여부      결석사유                  │
│  홍길동  37기   ● 출석        -                        │
│  김민서  38기   ✕ 결석       [알바 ▼]                  │
│  ...                                                │
├─────────────────────────────────────────────────────┤
│                          [저장 (비활성/활성)]          │
└─────────────────────────────────────────────────────┘
```

## 주요 동작 규칙

### 필터 (교구 / 팀 / 그룹)

| 선택 상태 | 표시 범위 |
|----------|----------|
| 교구만 선택 | 해당 교구 전체 인원 |
| 교구 + 팀 | 해당 팀 인원만 |
| 교구 + 팀 + 그룹 | 해당 그룹 인원만 |

- 교구: 1교구 / 2교구 / 3교구 (고정)
- 팀: 1팀 ~ 12팀 (고정)
- 그룹: 1그룹 ~ 4그룹 (고정)
- 팀은 교구 선택 후 활성화, 그룹은 팀 선택 후 활성화 (cascade)

### 주차 네비게이션

- 기본값: 앱 진입 시 **가장 최근 토요일** 자동 선택
- `< 이전` / `다음 >` 버튼으로 1주씩 이동
- **미래 주차는 이동 불가** → 다음 버튼 비활성(disabled)
- **과거 주차는 자유롭게 이동 가능** → 소급 입력 및 수정 지원

### 출석 입력

- `status`: 출석 / 결석 토글
  - 해당 주차에 `attendance_record`가 없는 교인은 기본값 **결석**으로 표시
- `absent_reason`: status가 **결석일 때만** Select 활성화
  - 선택지: 학교/학원, 회사, 알바, 가족모임, 개인일정, 아픔, 기타
  - status가 **출석으로 변경되면 absent_reason 자동 초기화(null)**

### 저장 버튼

- 초기 로드 데이터와 비교해서 **변경된 항목이 하나라도 있으면** 활성화
- 변경 없으면 **비활성(disabled)** 유지
- 저장 클릭 시:
  - 해당 주차에 기존 레코드가 없는 항목 → **POST** (신규 생성)
  - 기존 레코드가 있는 항목 → **PUT** (수정)
  - 변경된 항목만 API 전송 (전체 재전송 아님)
- 저장 성공/실패 → **Snackbar** 피드백

## DB 매핑 (`attendance_record`)

| UI 항목 | DB 컬럼 | 비고 |
|---------|--------|------|
| 주차 날짜 | `worship_date` | 해당 주 토요일 date |
| 교인 | `member_id` | member FK |
| 교구 스냅샷 | `gyogu` | 저장 시점 교구 번호 |
| 팀 스냅샷 | `team` | 저장 시점 팀 번호 |
| 그룹 스냅샷 | `group_no` | 저장 시점 그룹 번호 |
| 출석여부 | `status` | PRESENT / ABSENT |
| 결석사유 | `absent_reason` | 결석일 때만, 출석이면 null |

## 변경 이력 (추가 요구사항)

### v1.1 — 달력 Popover, 페이징, 이탈 경고

#### 주차 날짜 클릭 → 달력 Popover
- 주차 날짜 레이블 클릭 시 **MUI Popover** 형태의 달력 표시
- 달력은 월 단위로 표시, 월 이동 버튼(`< >`) 제공
- **토요일만 선택 가능** (파란색 강조), 미래 토요일은 disabled
- 선택 시 해당 날짜로 worshipDate 변경 후 Popover 닫힘

#### 페이징 처리
- 교구·팀 단위 조회 시 50명 이상 가능 → DataTable `pagination` prop 사용
- 기본 페이지 크기: **20명**
- 필터 또는 날짜 변경 시 **page 0으로 초기화**
- 번호 컬럼: `page * rowsPerPage + index + 1` 로 전체 순번 표시

#### 이탈 경고
- 미저장 변경사항이 있을 때 **브라우저 탭 닫기 / 새로고침** → `beforeunload` 이벤트로 경고
- 앱 내 라우터 이동 차단(`useBlocker`)은 현재 앱이 `<BrowserRouter>` 기반이라 미지원 (data router 전용 API), 추후 `createBrowserRouter` 마이그레이션 시 재도입 검토

### v1.2 — 직분 컬럼 및 임원단 필터

#### 직분 컬럼 추가
- 테이블 컬럼 순서: 번호 / 교구 / 팀 / 그룹 / **직분** / 이름 / 기수 / 출석여부 / 결석사유
- `member_profile.leader` (leader_id CSV) → `leader` 테이블 조인 → `leader_name` 목록으로 반환
- 한 교인이 **복수 직분** 보유 가능 (예: 팀장 + 임원단)
- 직분 값: `팀장` / `그룹장` / `PLT` / `새큼터` / `부서장` / `부팀장` / `새가족` / `임원단`
- 직분이 있으면 `Chip` 컴포넌트로 라벨 렌더링, 없으면 `-` 표시

#### 임원단 필터
- 교구 드롭박스에 **임원단** 옵션 추가 (1교구~3교구 외)
- 임원단 선택 시: `leaderNames.includes('임원단')` 인 교인 전체 표시 (교구 구분 없음)
- 임원단 선택 시: **팀 / 그룹 드롭박스 자동 비활성화**

#### API 데이터 구조 (직분 관련)
| UI 항목 | 출처 | 비고 |
|---------|------|------|
| 직분(들) | `member_profile.leader` → `leader.leader_name` | CSV → 배열로 변환하여 반환 |

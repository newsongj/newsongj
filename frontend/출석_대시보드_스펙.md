# 출석 대시보드 스펙 (AttendanceDashboard)

## 파일 위치
`src/apps/pages/AttendanceDashboard.tsx`

## 화면 목적
출석 관리(AttendancePage)에서 입력된 데이터를 기반으로 **출석 현황 집계 및 시각화**를 제공하는 대시보드.
필터링 조건 기반으로 실시간 데이터를 보여주며, 전 기간 대비 비교 없이 선택 기간의 데이터만 집계한다.

---

## 메뉴 위치
- 사이드바 `교적관리` 클릭 → 대시보드 페이지로 이동 (메인 랜딩)
- 교적관리 서브메뉴 첫 번째 항목: **출석 대시보드** (`/attendance-dashboard`)

## 주요 동작 규칙

### 필터 패널

| 필터 | 설명 |
|------|------|
| 기간 단위 | 주간 / 월간 / 연간 / 직접 입력 |
| 주간 선택 시 | `‹ 2026년 3월 15일 주 ›` 이전/다음 주 버튼. 미래 주 이동 불가 |
| 월간 선택 시 | 연도 Select + 월 Select |
| 연간 선택 시 | 연도 Select |
| 직접 입력 시 | 시작일 `<input type="date">` ~ 종료일 `<input type="date">` |
| 교구 | 전체 / 1교구 / 2교구 / 3교구 / 임원단 |
| 팀 | 전체 / 1~12팀 (교구 선택 후 활성화, cascade) |
| 임원단 선택 시 | 팀 드롭박스 비활성화 |
| 그룹 필터 | **미제공** (그룹 단위 필터링 불필요) |

> 모든 차트는 위 필터 조건을 공유하며 실시간으로 집계 데이터가 반영된다.
> 전 기간 대비 비교는 제공하지 않는다.

### KPI 카드 (4개)

| 카드 | 메인 값 | 보조 텍스트 | 설명 |
|------|--------|-----------|------|
| 기간 평균 출석 인원 | N명 | 전체 N명 기준 | 선택 기간 내 주차/월 평균 출석 인원 수 |
| 45기 출석 인원 | N명 | 전체 N명 중 | 필터 기간 내 45기 출석 인원 |
| 46기 출석 인원 | N명 | 전체 N명 중 | 필터 기간 내 46기 출석 인원 |
| 최다 출석 교구 | N교구 | N명 / N명 | 해당 기간 교구별 출석 인원 중 1위 교구 |

> 45기·46기는 현재 가장 최근 2개 기수 기준이며, API 연동 시 동적으로 최근 기수를 조회한다.

### 차트 구성

#### ① 출석 인원 추이 (LineChart)
- 드롭박스 없음 (상단 필터 기간 설정이 그대로 반영)
- x축: 날짜/주차/월/연도 (기간 단위에 따라 자동 변경)
- y축: 인원 수(명)
- 출석 인원 단일 Line (파란 실선)
- 차트 description에 현재 필터 기간 텍스트 표시

#### ② 차원별 출석 인원 (BarChart)
- 차트 우측 상단 드롭박스로 차원 선택: 교구별 / 팀별 / 기수별 / 성별
- x축: 차원값, y축: 인원 수(명)
- 출석 인원 단일 Bar (파란색)
- **직분별 차원은 제공하지 않음** (집계 복잡도 및 중복 카운트 이슈)

#### ③ 결석사유 분포 (Donut PieChart)
- 드롭박스 없음
- 결석 인원 중 사유별 비중 표시
- 사유: 학교/학원, 알바, 회사, 개인일정, 가족모임, 아픔, 기타
- 차트 우측에 범례 (색상 + 사유명 + 인원수)

#### ④ 교구별 출석 현황 (Stacked BarChart)
- 드롭박스 없음
- 교구별(1~3교구 + 임원단) 출석/결석 인원 수 Stacked Bar
- 파란색(출석), 빨간색(결석)

## 집계 설계 (핵심 원칙)

### Slowly Changing Dimension 처리

**원칙: 당시 시점 스냅샷 기준 집계**

| 집계 기준 | 사용 데이터 | 이유 |
|----------|-----------|------|
| 날짜 범위 내 교구/팀 집계 | `attendance_record.gyogu/team/group_no` | 이동 전·후 소속 이력을 정확하게 반영 |
| 연간 집계 | `member_profile.year` 기준 조인 | 연 단위 스냅샷 활용 |
| 기수/성별 | `member.generation/gender` | 불변 값이므로 현재 기준 JOIN 가능 |
| 직분 | `member_profile.leader` → `leader.leader_name` | 연 단위 스냅샷 기준 |

### 날짜 범위가 연도를 걸칠 때
- `attendance_record` 스냅샷 기준으로 통일
- `member_profile`은 각 연도별로 분리 조회 후 병합

### API 요청 파라미터 (예상)
```
GET /api/attendance/dashboard
  ?start_date=2026-01-01
  &end_date=2026-03-15
  &gyogu=1          (optional)
  &team=2           (optional)
  &dimension=gyogu  (교구별/팀별/기수별/성별)
```

## 확정된 설계 결정 사항

1. **집계 기준**: 당시 시점 스냅샷 (`attendance_record` 스냅샷 필드 우선)
2. **전 기간 대비 비교 없음**: 필터링 조건 기반 실시간 데이터만 표시
3. **그룹 필터 없음**: 교구 → 팀 2단계 cascade로 충분
4. **임원단**: 교구 필터에 별도 옵션으로 추가, 선택 시 팀 드롭박스 비활성화
5. **KPI 카드**: 기간 평균 출석 인원 / 45기 출석 인원 / 46기 출석 인원 / 최다 출석 교구
6. **출석 인원 추이·차원별**: 출석 인원만 표시 (결석 제거)
11. **차원별 출석 인원 차원 옵션**: 교구별 / 팀별 / 기수별 / 성별 4개. 직분별은 복수 직분 중복 카운트 이슈로 제외
7. **결석사유 분포·교구별 출석 현황·출석 인원 추이**: 차트 자체에 드롭박스 없음 (상단 필터 공유)
8. **교적관리 클릭 시**: `/attendance-dashboard`로 자동 이동 (사이드바 부모 path 설정)
9. **기존 DashboardPage**: 건드리지 않음 (별도 서비스용 대시보드)
10. **직접 입력 날짜 범위**: 1차 구현 포함 (`<input type="date">` 사용)

## API 집계 수식 (백엔드 참고용)

### 공통 사전 정의

```
-- 필터 파라미터
:start_date  -- 기간 시작일 (토요일)
:end_date    -- 기간 종료일 (토요일)
:gyogu       -- 교구 번호 (optional, 임원단이면 별도 처리)
:team        -- 팀 번호 (optional)
:generation  -- 기수 (optional, KPI용)
:dimension   -- 차원 (gyogu|team|generation|gender)

-- 전체 인원 기준
:  attendance_record에 해당 기간 내 worship_date 레코드가 존재하는 member
   (PRESENT + ABSENT 포함, 해당 기간 출석 명단에 올라간 인원 = 전체 인원)

-- 임원단 leader_id
:imonjang_id = SELECT leader_id FROM leader WHERE leader_name = '임원단'
```

---

### 기간 단위 → start_date / end_date 변환

| 기간 단위 | start_date | end_date |
|----------|-----------|---------|
| 주간 | 선택한 토요일 | 선택한 토요일 |
| 월간 | 해당 월의 첫 번째 토요일 | 해당 월의 마지막 토요일 |
| 연간 | 해당 연도의 첫 번째 토요일 | 해당 연도의 마지막 토요일 |
| 직접 입력 | 사용자 입력 시작일 | 사용자 입력 종료일 |

> `worship_date`는 항상 토요일이므로 BETWEEN 조건으로 자연스럽게 필터링됨

---

### 임원단 필터 처리

교구 필터로 `임원단`을 선택한 경우, `attendance_record.gyogu`가 아닌 **`member_profile.leader`** 기준으로 필터링

```sql
-- 임원단 멤버 서브쿼리 (worship_date 연도 기준 member_profile 조인)
WITH imonjang_members AS (
  SELECT DISTINCT ar.member_id
  FROM attendance_record ar
  JOIN member_profile mp
    ON ar.member_id = mp.member_id
    AND YEAR(mp.year) = YEAR(ar.worship_date)
  WHERE FIND_IN_SET(:imonjang_id, mp.leader) > 0
    AND ar.worship_date BETWEEN :start_date AND :end_date
)
-- 이후 쿼리에서 attendance_record JOIN imonjang_members 조건 추가
```

### KPI 카드

#### ① 기간 평균 출석 인원

> 기간 내 worship_date별 출석 인원 수의 평균

```sql
SELECT ROUND(AVG(daily_present), 0) AS avg_present_count
FROM (
  SELECT worship_date, COUNT(*) AS daily_present
  FROM attendance_record
  WHERE worship_date BETWEEN :start_date AND :end_date
    AND status = 'PRESENT'
    [AND gyogu = :gyogu]       -- 교구 필터 적용 시
    [AND team  = :team]        -- 팀 필터 적용 시
  GROUP BY worship_date
) sub
```

> 전체 N명 기준 = 같은 조건에서 status 무관 COUNT (PRESENT + ABSENT)

```sql
SELECT ROUND(AVG(daily_total), 0) AS avg_total_count
FROM (
  SELECT worship_date, COUNT(*) AS daily_total
  FROM attendance_record
  WHERE worship_date BETWEEN :start_date AND :end_date
    [AND gyogu = :gyogu]
    [AND team  = :team]
  GROUP BY worship_date
) sub
```

#### ② 45기 / 46기 출석 인원

> 기간 내 해당 기수의 출석 인원 수 (주간이면 해당 주, 월간/연간이면 평균)

```sql
-- 예: 45기
SELECT COUNT(*) AS present_count
FROM attendance_record ar
JOIN member m ON ar.member_id = m.member_id
WHERE ar.worship_date BETWEEN :start_date AND :end_date
  AND ar.status = 'PRESENT'
  AND m.generation = 45
  [AND ar.gyogu = :gyogu]
  [AND ar.team  = :team]

-- 전체 45기 인원 (해당 기간 출석 명단 기준)
SELECT COUNT(DISTINCT ar.member_id) AS total_count
FROM attendance_record ar
JOIN member m ON ar.member_id = m.member_id
WHERE ar.worship_date BETWEEN :start_date AND :end_date
  AND m.generation = 45
  [AND ar.gyogu = :gyogu]
  [AND ar.team  = :team]
```

> 기간이 주간이 아닌 경우(월간/연간) present_count도 AVG 처리 권장

---

#### ③ 최다 출석 교구

> 기간 내 교구별 출석 합계 중 가장 많은 교구

```sql
SELECT gyogu, COUNT(*) AS present_count
FROM attendance_record
WHERE worship_date BETWEEN :start_date AND :end_date
  AND status = 'PRESENT'
GROUP BY gyogu
ORDER BY present_count DESC
LIMIT 1
```

### 차트

#### ① 출석 인원 추이 (Line Chart)

> x축: worship_date, y축: 해당 날짜 출석 인원 수

```sql
SELECT worship_date, COUNT(*) AS present_count
FROM attendance_record
WHERE worship_date BETWEEN :start_date AND :end_date
  AND status = 'PRESENT'
  [AND gyogu = :gyogu]
  [AND team  = :team]
GROUP BY worship_date
ORDER BY worship_date ASC
```

---

#### ② 차원별 출석 인원 (Bar Chart)

**dimension = gyogu (교구별)**
```sql
SELECT gyogu AS dim, COUNT(*) AS present_count
FROM attendance_record
WHERE worship_date BETWEEN :start_date AND :end_date
  AND status = 'PRESENT'
GROUP BY gyogu
ORDER BY gyogu ASC
```

**dimension = team (팀별)**
```sql
SELECT team AS dim, COUNT(*) AS present_count
FROM attendance_record
WHERE worship_date BETWEEN :start_date AND :end_date
  AND status = 'PRESENT'
  [AND gyogu = :gyogu]
GROUP BY team
ORDER BY team ASC
```

**dimension = generation (기수별)**
```sql
SELECT m.generation AS dim, COUNT(*) AS present_count
FROM attendance_record ar
JOIN member m ON ar.member_id = m.member_id
WHERE ar.worship_date BETWEEN :start_date AND :end_date
  AND ar.status = 'PRESENT'
  [AND ar.gyogu = :gyogu]
  [AND ar.team  = :team]
GROUP BY m.generation
ORDER BY m.generation ASC
```

**dimension = gender (성별)**
```sql
SELECT m.gender AS dim, COUNT(*) AS present_count
FROM attendance_record ar
JOIN member m ON ar.member_id = m.member_id
WHERE ar.worship_date BETWEEN :start_date AND :end_date
  AND ar.status = 'PRESENT'
  [AND ar.gyogu = :gyogu]
  [AND ar.team  = :team]
GROUP BY m.gender
```

> ⚠️ `dimension = leader (직분별)` 은 한 사람이 복수 직분을 가질 수 있어 중복 카운트 이슈가 있으므로 **화면에서 제공하지 않음** (확정 결정 사항 11번 참고).

---

#### ③ 결석사유 분포 (Donut Chart)

```sql
SELECT absent_reason, COUNT(*) AS absent_count
FROM attendance_record
WHERE worship_date BETWEEN :start_date AND :end_date
  AND status = 'ABSENT'
  AND absent_reason IS NOT NULL
  [AND gyogu = :gyogu]
  [AND team  = :team]
GROUP BY absent_reason
ORDER BY absent_count DESC
```

---

#### ④ 교구별 출석 현황 (Stacked Bar Chart)

```sql
SELECT
  gyogu,
  SUM(CASE WHEN status = 'PRESENT' THEN 1 ELSE 0 END) AS present_count,
  SUM(CASE WHEN status = 'ABSENT'  THEN 1 ELSE 0 END) AS absent_count
FROM attendance_record
WHERE worship_date BETWEEN :start_date AND :end_date
GROUP BY gyogu
ORDER BY gyogu ASC
```

### 주의사항

| 항목 | 내용 |
|------|------|
| 교구/팀/그룹 스냅샷 | `attendance_record.gyogu/team/group_no` 사용 (기록 당시 소속 기준) |
| 기수/성별 | `member.generation/gender` JOIN (불변 값) |
| 직분 | 차원별 집계 미제공. 임원단 필터에서만 `member_profile.leader` CSV → `FIND_IN_SET` 사용 |
| 삭제된 교인 | `member.deleted_at IS NULL` 조건 불필요 — `attendance_record`에 이미 기록된 데이터 기준 집계 |
| 연도 경계 | 날짜 범위가 연도를 걸칠 경우 `YEAR(ar.worship_date)`로 각 레코드에 맞는 `member_profile.year` 조인 |
| 임원단 필터 | `attendance_record.gyogu`가 아닌 `member_profile.leader` 기반 필터링 |
